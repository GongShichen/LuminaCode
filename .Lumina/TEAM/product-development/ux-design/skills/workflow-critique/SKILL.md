---
name: Workflow Critique
description: Critique user workflow and interaction sequence across product surfaces.
when-to-use: Use when designing or reviewing onboarding, task flows, command flows, forms, navigation, multi-step operations, or recovery paths.
user-invocable: false
context: inline
---

Review:

- Discoverability.
- Number of steps.
- User control during long-running work.
- Error and recovery clarity.
- Whether the flow matches existing product conventions.
- Primary user goal, interruptions, cancellation, undo/redo, confirmation, and safe retry behavior.
- Visibility of progress, state, ownership, and next action.
- Accessibility, localization, content density, and cognitive load.
